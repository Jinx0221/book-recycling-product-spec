SYSTEM OVERVIEW
系统架构总览
本系统共包含 5 个端口，覆盖图书回收全链路。书籍来源分为两大类型：图书单本回收（C端用户扫码录入，有明确书籍清单）和称重批次回收（大宗/现场建单，无逐本清单），两种类型在仓库环节走不同流程，但最终上架操作统一。

C1-P01
书籍扫码识别页
用户扫描书籍条形码/ISBN，系统自动识别并拉取书籍信息与预估价，确认后加入回收列表。
  • 调用相机扫描条形码，或手动输入ISBN
  • 展示：书籍封面、书名、作者、出版社、ISBN
  • 展示单本预估回收价（元）
  • 录入数量（默认1，可增减）
  • 「加入回收列表」按钮
  • 支持无间断连续扫码
  • 底部悬浮入口：当前已录入书籍数量

C1-P02
待回收书籍列表页
汇总已录入书籍，支持修改数量和删除，满足最低回收金额后可进入下一步。
  • 列表展示：封面缩略图、书名、预估单价、数量
  • 每条书籍支持修改数量（+/-）
  • 每条书籍支持删除（左滑或删除按钮）
  • 底部展示：书籍总数、预估总价
  • 「继续扫码」返回扫码页

⚠️ 金额门槛：预估总价低于系统预设金额时，「下一步」置灰，提示「预估价格不足 ¥XX，暂不支持提交」。

SHARED MODULE · 公共复用
现场建单模块（回收员 & 站点管理员共用）
通用现场称重建单能力，回收员和站点管理员均可调用。品类（废旧书籍各等级、废旧衣物等）由后台统一维护，选择品类后自动带出单价，支持现场微调。此模块产生的订单属于称重批次，进入仓库后走「分拣上架」流程。
原 C端大宗回收
客户自助扫码 → 灯色区间 → 按重量 → 自行提单，无需工作人员操作
保持独立，客户自助模式不变
原 站点直收（无订单）
管理员扫码查价 → 按区间称重 → 生成二维码 → 学生扫码确认
整合进现场建单模块，不再单独维护
🆕 现场建单（本模块）
回收员/管理员操作 → 多品类称重 → 单价可微调 → 生成确认码 → 用户扫码确认
扩展：废书分级 + 衣物 + 其他品类，两端共用同一套页面
🔵 回收员端触发场景
  • 上门核验时，部分书籍不符合图书回收标准，被剔出后当场按废品称斤建单
  • 用户同时有废旧衣物等其他品类，一并现场建单
  • 现场单与原图书回收订单独立存在，自动关联同一上门任务
🟢 站点管理员端触发场景
  • 学生送来混杂物品（图书+衣物+其他），管理员综合建单
  • 学生未在C端提前下单，直接到站点的散客
  • 复用原「站点直收无订单」流程，品类扩展至多品类

FIELD-P01 · 共用
现场建单 · 品类录入页
核心建单页面。操作者逐条添加回收品类，选择品类后带出后台配置的单价，支持现场微调，录入重量后加入清单，支持添加多个品类条目。
  • 品类选择：下拉选择，列表由后台维护（废旧书籍·红区 / 黄区 / 绿区 / 废旧衣物·普通 / 品牌 / 其他等）
  • 单价带出：选择品类后自动填入后台配置的参考单价（元/斤 或 元/kg）
  • 单价可调：支持手动修改，修改后标黄色高亮「已调整」，调整范围受后台上下限约束
  • 录入该品类重量（支持小数，单位与品类配置一致）
  • 小计实时计算：单价 × 重量 = 小计
  • 「加入清单」将条目加入本次回收清单
  • 「继续添加品类」，新增另一条目
  • 清单区展示已添加条目，支持删除/修改
  • 底部汇总：品类数 N 种 / 总重量 / 合计金额 ¥Y
  • 「生成确认单」跳转至确认码页
📋 回收员预填逻辑：从上门取书页「剔出书籍建废品单」入口跳转时，系统自动预填对应区间品类并关联上门任务ID，回收员只需确认/调整重量即可。

现场建单的品类和单价由后台管理系统统一维护，前端动态拉取，无需发版更新。

B-END · WAREHOUSE
B端 · 仓库管理员
仓库管理员是书籍进入销售体系的最后关口。根据批次类型走两套不同流程：图书批次（来自C端图书回收）做拆包逐本核验，追责精确到本；称重批次（来自大宗回收/现场建单）入库轻量化，仓管直接分拣挑出好书上架，废品无需系统记录。两种批次上架操作完全统一。
📗 图书批次流程（来自C端图书回收订单）
1
扫转运码 → 调出书籍清单（含订单号、书名、数量），以批次维度展示
2
清点包装 → 对照订单逐包确认，发现运损/数量不符可标记，系统追溯责任人
3
完成入库 → 书籍状态变「已入库待核验」
4
拆包逐本核验 → 检查外观/新旧/笔记/正版，发现问题可标记追责
5
核验通过 → 书籍状态变「待上架」，进入上架流程
📦 称重批次流程（来自大宗回收 / 现场建单）
1
扫转运码 → 调出批次信息（来源供货方、价格区间、重量、照片），无逐本清单
2
确认入库 → 批次状态变「已入库待分拣」，无需清点包装环节
3
直接分拣 → 仓管拆包翻书，把值得上架的书挑出来，废品直接归废品堆，系统不记录废品
4
上架 → 挑出的好书直接进上架流程，扫ISBN生成货号，自动关联来源批次号
✅ 设计原则：称重批次的追责单元是「批次」而非「本」。废品占比高由供货方口头反馈或业务层处理，系统不需要强制记录每一斤的去向，保持流程轻量。
🔑 上架操作完全统一（两种批次一致）：仓管拿到一本书 → 扫ISBN → 选货架 → 系统生成唯一货号（格式：A-3-L-001，即A货架3层左侧001号位）→ 调用蓝牙热敏打印机打印标签 → 贴书脊。同一ISBN可对应货架上多本书，每本货号唯一。书籍档案自动记录来源字段（图书回收订单号 or 称重批次号），互不影响。

B3-P06
书籍上架操作页（两种批次统一）
所有待上架书籍（图书批次核验通过 + 称重批次分拣挑出）在此统一上架。扫描ISBN后系统并行查询孔网最新价格与系统内已有商品链接，由上架人员人工判断并选择处理路径，最终生成全局唯一货号，调用蓝牙打印机打印标签。
① 扫码 — 并行查询两个数据源
  • 扫描书籍ISBN条形码，系统立即并行发起两个查询：
    孔网接口（实时）
    返回该ISBN的最新市场参考价、书名、基础信息
    系统内部查询
    返回该ISBN下所有已有商品链接（名称、图片、在售价格、当前库存数），可能返回0条、1条或多条
② 展示结果 — 上架人员判断选择路径
路径一 · 首次上架
系统内无该ISBN商品记录
展示「首次上架」提示 + 孔网参考价

操作：确认/修改书名 → 确认/修改价格 → 上传图片（必填）

提交：创建新商品链接 → 货号生成 → 库存=1
路径二 · 复用已有商品
系统内有1条或多条商品记录
展示所有已有商品链接列表供选择

上架人员从列表中点选「是同一本书」

• 在售价 = 孔网价：直接确认上架
• 在售价 ≠ 孔网价：显示价差提示，可勾选「同步价格」

提交：复用商品链接 → 货号生成 → 库存+1
[勾选同步价格] 后台触发价格更新任务
路径三 · 新品上架（ISBN已有记录但书不同）
系统内有记录，但上架人员判断不是同一本书
（版本/印次/封面不符）

点击商品列表底部「按新品上架」按钮

操作：确认/修改书名 → 确认/修改价格 → 上传新图片（必填）

提交：创建全新独立商品链接 → 货号生成 → 库存=1
（不合并、不影响原有商品库存）
③ 已有商品链接列表展示规则
  • 同一ISBN下若有多条商品链接（因多次路径三新增），全部展示在列表中，每条独立可选
  • 每条商品链接展示：商品图片缩略图、书名、当前在售价格、当前库存数量
  • 列表底部始终保留「按新品上架」入口，不论已有多少条记录
  • 系统不限制同一ISBN下商品链接数量上限
④ 价差提示与同步价格逻辑
  • 触发条件：选中某条已有商品后，其在售价格与本次孔网参考价不相等时，立即展示价差提示（无阈值，只要不同就提示）
  • 提示样式：「孔网参考价 ¥X，当前在售价 ¥Y，存在价差」+ 勾选框「同步至孔网价格」
  • 「同步价格」默认不勾选，上架人员主动勾选才触发
  • 勾选后提交：本次上架正常完成（库存+1）的同时，后台异步触发价格更新任务，将该商品链接在售价更新为本次孔网参考价
  • 不勾选：正常上架，当前在售价格不变
⑤ 货架选择与打印标签（三条路径均一致）
  • 商品链接确认后，选择目标货架（显示货架号及剩余容量，如 A-3货架 剩余87/130位）
  • 系统生成全局唯一货号（格式：货架-层-左右-序号，如 A-3-L-001）
  • 立即调用蓝牙热敏打印机打印标签（货号 + 书名简称 + ISBN条形码）
  • 标签贴至书脊，将书放入对应货架位置
  • 货架满时提示「已达最大容量，请切换货架」，可临时新建货架
  • 上架完成：书籍状态更新为「已上架」，记录货号、货架位置、商品链接ID、来源批次号
✅ 货号与商品链接的关系：货号是每本物理书在仓库的唯一位置标识，商品链接是面向销售侧的商品档案（含价格、图片、库存）。两者独立存在：同一商品链接可对应N个货号（N本同书），同一货号只对应一本物理书。库存数 = 该商品链接下所有有效货号的数量。
📌 来源字段透明：每本书上架后档案自动记录来源——图书批次显示「来源：订单 #XXXXXX」，称重批次显示「来源：称重批次 #XXXXXX」。ISBN重复不影响任何逻辑，货号全局唯一。
