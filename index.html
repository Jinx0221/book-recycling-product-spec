C端 · 散客 / 大学生
面向普通大学生和个人用户的回收入口，支持单本扫描录入、在线提交订单、选择回收方式（上门/送站点），以及订单管理全流程。

→
待回收列表
→
选择回收方式
→
填写信息/拍照
→
提交订单
→
订单跟踪
→
交易完成
📷
C1-P01
书籍扫码识别页
用户通过摄像头扫描书籍条形码/ISBN，系统自动识别并拉取书籍信息与预估价，用户确认后添加至回收列表。
  • 调用相机，扫描书籍条形码或手动输入ISBN
  • 展示书籍封面、书名、作者、出版社、ISBN
  • 展示单本预估回收价（元）
  • 录入回收数量（默认1，可增减）
  • 点击「加入回收列表」按钮
  • 支持无间断连续扫码，扫下一本
  • 底部悬浮展示当前已录入书籍数量入口
📋
C1-P02
待回收书籍列表页
汇总已扫码录入的全部书籍，用户可对列表进行修改，完成后进入下一步订单确认流程。
  • 列表展示：书籍封面缩略图、书名、预估单价、数量
  • 每条书籍支持修改数量（+/-）
  • 每条书籍支持删除（左滑或删除按钮）
  • 底部展示书籍总数、预估总价
  • 底部按钮「继续扫码」返回扫码页
  • 底部按钮「下一步」进入订单确认（预估价满足门槛才可点击，否则灰色+提示）
⚠️ 门槛控制：预估回收总价低于系统预设金额时，「提交」按钮置灰并显示「预估价格不足 ¥XX，暂不支持回收」提示。
🔄
C1-P03
回收方式选择页
用户选择本次回收的服务方式，不同方式跳转至对应的信息填写页。
  • 三个选项卡：上门回收 / 送至站点 / 快递邮寄
  • 「快递邮寄」显示为禁用状态，附说明文字「即将开放」
  • 「上门回收」展示服务说明：免费上门、核验后结算
  • 「送至站点」展示说明：自行送达校内站点
  • 选择后点击「下一步」进入对应填写页
🏠
C1-P04
上门回收信息填写页
选择「上门回收」后填写具体取书地址、预约时间，并上传书籍整体照片，完成订单提交前的最后信息录入。
  • 选择学校（下拉/搜索，仅显示已入驻校园列表）
  • 填写详细地址（如：知行楼5楼102室）
  • 预约上门时间：日期选择 + 时段选择（如08:00-10:00、10:00-12:00等）
  • 上传书籍整体照片（拍照或相册，至少1张）
  • 展示书籍数量、预估总价汇总
  • 点击「提交订单」生成上门回收订单
📸 拍照要求提示：页面内展示拍照参考示例图，提示用户将所有书籍整齐摆放后拍照。
📍
C1-P05
送至站点信息确认页
选择「送至站点」后，用户选择目标站点，系统展示站点详情，用户确认后提交订单，获得二维码凭证。
  • 选择学校（仅显示已入驻校园）
  • 选择站点，展示站点名称、位置、营业时间
  • 地图/文字方式展示站点位置
  • 展示书籍清单及预估总价
  • 点击「提交并生成凭证」提交订单
🔑 凭证说明：提交后立即跳转至核验码展示页，用户送书时需出示此码供站点管理员扫描核验。
📱
C1-P06
订单核验码展示页
送至站点模式下，用户将书籍送到站点时出示本页二维码，供站点管理员扫码核验订单信息。
  • 居中展示大尺寸订单二维码
  • 二维码下方展示订单号、书籍数量
  • 展示站点名称、站点营业时间
  • 提示文字：「请将书籍送达站点后，让工作人员扫描此码」
  • 按钮：「查看订单详情」跳转至订单详情页
📦
C1-P07
订单列表页
用户查看所有历史和进行中的回收订单，支持按状态筛选，对未完成订单可执行取消操作。
  • Tab筛选：全部 / 进行中 / 已完成 / 已取消
  • 每条订单展示：订单号、回收方式标签、书籍数量、预估金额、当前状态
  • 未完成订单可点击「取消订单」并二次确认
  • 点击订单进入订单详情页
待回收员上门 核验中 已完成 已取消
🧾
C1-P08
订单详情页
展示单个订单的完整信息，包括书籍明细、回收方式详情、状态流转记录，以及核验结果通知。
  • 订单号、提交时间、回收方式
  • 书籍列表：书名、数量、预估单价
  • 上门回收：展示取书地址、预约时段
  • 送至站点：展示站点名称、位置文字、营业时间
  • 状态时间轴：已提交→已派单→回收员已到达→核验完成→交易完成
  • 核验通过书籍照片（核验完成后显示）
  • 最终回收金额（交易完成后显示）
  • 未完成状态显示「取消订单」按钮

C-END · BULK
C端 · 废品站 / 大宗回收
面向废品站、大批量回收商，通过扫码获取书籍价格区间（灯色标识），按重量录入，快速提交大宗回收订单，无需逐本录入。
大宗回收核心交互流程
扫码查询区间
→
按区间录入重量
→
加入待回收列表
→
填写地址/联系方式
→
提交订单
→
订单管理
💡 价格区间灯色说明：大宗回收不展示具体金额，只展示价格区间的颜色标识：
🔴 低价区 2-4元/kg 🟡 中价区 5-8元/kg 🟢 高价区 ≥9元/kg
📷
C2-P01
书籍扫码查价页（大宗）
废品站用户扫描书籍条码，系统返回该书所属的价格区间灯色（不展示具体价格），用户录入该类书籍重量并加入列表。
  • 调用相机扫描书籍条形码/ISBN
  • 展示书籍名称（简略）
  • 以大灯颜色方式展示价格区间（红/黄/绿）
  • 不展示具体金额，只显示区间描述（低/中/高）
  • 录入该区间书籍重量（千克，支持小数）
  • 点击「加入列表」将该条目加入待回收列表
  • 支持连续扫码录入多类书籍
⚖️
C2-P02
待回收列表页（大宗）
汇总已录入的各价格区间书籍及对应重量，支持修改删除，底部显示各区间汇总重量。
  • 按价格区间（红/黄/绿）分组展示
  • 每组展示灯色标识 + 书籍条目名称 + 重量（kg）
  • 每条支持修改重量、删除条目
  • 底部汇总各区间总重量（如：🔴 12kg / 🟡 30kg / 🟢 5kg）
  • 「继续扫码」返回扫码页
  • 「下一步」进入信息填写页
📝
C2-P03
上门回收信息填写页（大宗）
大宗用户填写取货地址和联系方式，确认各区间汇总重量后提交订单。
  • 填写取货详细地址
  • 填写联系人姓名、联系电话
  • 展示各区间汇总重量（不展示金额）
  • 预约上门时间（日期+时段选择）
  • 点击「提交订单」生成大宗回收单
📋
C2-P04
订单列表页（大宗）
大宗用户查看历史订单列表，支持查看详情、取消进行中订单。
  • Tab筛选：全部 / 进行中 / 已完成 / 已取消
  • 每条订单：订单号、提交时间、各区间总重量、状态
  • 进行中订单支持「取消」操作
  • 点击订单跳转至订单详情页
🧾
C2-P05
订单详情页（大宗）
展示大宗回收订单的完整信息，包含各区间书籍重量、取货信息、订单状态等。
  • 订单号、提交时间
  • 取货地址、联系人、联系电话
  • 各区间重量汇总（红/黄/绿灯色标识）
  • 状态时间轴展示
  • 最终结算金额（完成后显示，含各区间单价×重量）
  • 工作人员上传的书籍照片（完成后显示）
✅
C2-P06
站点直收确认单页（复用）
站点管理员操作生成确认单后，学生扫码拉起此页面，查看分类重量与回收总价，点击确认回收完成交易。此页面逻辑与大宗回收交互模式复用。
  • 展示各价格区间分类（A类/B类/C类）
  • 每类展示：回收单价（元/kg）+ 称重重量 + 小计
  • 展示回收总价合计（如：A类9元×8kg + B类5元×3kg = 87元）
  • 「确认回收」按钮，用户确认后交易进入待上传照片状态
  • 站点管理员端随后上传照片，完成交易
🔁 交互复用说明：此页由C端用户扫描管理员生成的二维码后打开，展示方式与大宗回收的「按区间+重量+总价」格式一致。

B-END · RECYCLER
B端 · 回收员（校内兼职）
回收员为校内招聘的兼职大学生，通过扫站点管理员二维码注册，每日上线接单并填写可接单时段，按系统派单执行上门取书、核验书籍、交接站点等操作。
回收员工作流程
注册入驻
→
每日上线+填写接单参数
→
系统派单接收
→
上门取书
→
核验/移除不合格书籍
→
打包上传照片
→
送至站点交接
→
完成结佣
🧑‍🎓
B1-P01
注册页（扫站点码入驻）
新回收员通过扫描站点管理员出示的专属小程序码后，进入此注册页填写个人信息完成注册，并自动关联至该站点。
  • 页面顶部展示所属站点名称（来自扫码信息）
  • 填写：姓名、联系电话、学院、班级、学号
  • 上传学生证或身份证照片（用于身份验证）
  • 勾选服务协议
  • 点击「提交注册」
  • 提交后显示「注册成功，等待管理员审核」状态页
🟢
B1-P02
上线接单页
回收员每日打开小程序后进行「上线」操作，填写今日可接单数量和时间区间，系统据此进行派单匹配。
  • 展示当前状态：「未上线」状态（红色）
  • 填写今日可接单数量（如：3单）
  • 选择可接单时间区间（多选时段，如：09:00-11:00、14:00-17:00）
  • 点击「上线接单」切换为「已上线」状态（绿色）
  • 上线后显示「今日接单状态」摘要：可接数/已接数
  • 入口跳转至「修改接单参数」
⏰ 灵活时段：时段选择支持多选，解决学生课表不规律问题。上线后可在个人中心随时修改今日接单参数。
📬
B1-P03
收书任务列表页
展示系统派给该回收员的所有上门取书任务，按预约上门时间排序，以颜色状态提醒紧急程度。
  • 按预约上门时间升序排列
  • 每条任务展示：取书地址、书籍数量、预约时段、倒计时
  • 紧急状态色标（距预约时间 <30min → 红色 / <1h → 黄色 / >1h → 绿色）
  • Tab筛选：待上门 / 核验中 / 已完成
  • 点击任务进入任务详情页
🔴 <30min 紧急 🟡 <1h 留意 🟢 >1h 正常
🚪
B1-P04
上门取书操作页
回收员到达用户地址后，点击「已到达」开始核验流程，逐一核验书籍，移除不合格书籍，上传照片完成回收。
  • 展示用户信息：取书地址、预约时段、联系方式
  • 展示用户上传的书籍整体照片（供核验参考）
  • 点击「已到达」开始核验（触发状态变更）
  • 书籍核验列表：每本展示书名、预估价、核验状态
  • 对不合格书籍可点击「移除」（需选择原因：破损/笔记过多/非正版等）
  • 核验完成后上传：通过核验书籍照片 + 打包照片（各至少1张）
  • 点击「完成回收」结束本单，C端用户订单更新为「交易完成」
📦
B1-P05
交接站点页（出示交接码）
回收员将所有打包书籍运送至站点时，出示交接码，由站点管理员扫码确认接收，完成交接流程。
  • 展示本次待交接的订单数量、总书籍数
  • 居中展示交接二维码（含回收员ID+订单批次信息）
  • 说明文字：「请出示此码，由站点管理员扫描确认接收」
  • 扫码确认后页面更新为「交接成功」状态
👤
B1-P06
个人中心页
回收员查看个人信息、修改今日接单参数、查看历史任务统计和结佣数据。
  • 展示：姓名、学院班级、所属站点、注册时间
  • 今日接单状态卡片：可修改接单数、时段（仅上线状态下可改）
  • 「下线」按钮：停止今日接单
  • 数据统计：本月已完成单数 / 本月结佣金额
  • 历史取送任务单列表入口
  • 结佣明细记录入口（按单结算详情）
💰
B1-P07
结佣明细页
回收员查看每笔已完成任务对应的佣金结算详情，了解收入来源和扣款记录。
  • 月度总计佣金金额
  • 按时间倒序列出每笔结算记录
  • 每条记录：订单号、完成时间、书籍数量、佣金金额
  • 如有扣款（如因核验问题被追责），展示扣款原因及金额
  • 支持月份筛选

04
B-END · STATION ADMIN
B端 · 站点管理员
站点管理员驻守校内直营站点，负责管理回收员、接收回收员交接书籍、处理学生自送订单（扫码已有订单/现场创建新订单），并将书籍统一转运至仓库。
站点管理员多场景操作流程
生成注册码（招募回收员）
→
扫回收员交接码（接收打包书）
→
扫学生核验码（已有订单）
→
现场称重创建订单（无订单学生）
→
转运出库（出示转运入库码）
🏠
B2-P01
站点首页 / 工作台
管理员进入后的主工作台，展示今日各业务数据概览，提供快捷操作入口。
  • 站点名称、今日日期
  • 数据卡片：今日已接收订单数 / 待转运单数 / 站点回收员在线数
  • 快捷操作：「扫码接收」「现场创建订单」「查看待转运」「管理回收员」
  • 消息通知：回收员交接到站提醒、系统通知
👥
B2-P02
回收员管理页
管理员查看本站点所有已注册的回收员，可生成招募二维码供新回收员扫码注册，也可对已有回收员进行管理操作。
  • 展示本站点回收员列表：姓名、学院班级、注册时间、今日状态（在线/离线）
  • 每个回收员可查看详情（注册信息、本月完成单数）
  • 「踢出站点」操作（需二次确认）
  • 顶部按钮：「生成招募码」——展示可供新人扫码注册的专属二维码
  • 待审核新回收员列表（如注册需审核）
📲
B2-P03
扫码操作页（多功能入口）
管理员使用此页进行所有扫码操作：扫回收员交接码（接收打包书籍）或扫学生核验码（处理学生自送订单）。
  • 扫码类型选择：「接收回收员交接」 / 「核验学生订单」
  • 调用相机扫描对应二维码
  • 扫码后跳转至对应确认页面
🤝
B2-P04
回收员交接确认页
扫描回收员出示的交接码后，管理员查看本次交接的订单汇总信息，确认接收后系统完成交接记录，书籍进入待转运列表。
  • 展示回收员姓名、交接时间
  • 本次交接订单列表：订单号、书籍数量、回收金额、订单照片缩略图
  • 汇总：共N单 / 共M本书 / 合计金额
  • 「确认接收」按钮：点击后订单状态更新，进入待转运列表
  • 如有异常可备注说明
✅
B2-P05
学生订单核验页（已有订单）
学生自送书籍至站点并出示二维码后，管理员扫码调出订单，核验书籍，完成站点直收流程。
  • 展示用户信息（头像、昵称）和订单信息
  • 书籍明细列表：书名、数量、预估价
  • 核验操作：对不符合标准的书籍点击「移除」（选择原因）
  • 核验完成后上传：通过核验书籍照片 + 打包照片
  • 点击「完成回收」：系统更新C端订单状态为「交易完成」
🆕
B2-P06
现场创建回收单页（无订单学生）
学生未提前在C端下单，管理员现场扫码查询书籍价格区间，按区间称重分类，创建回收确认单，学生扫码确认后完成交易。
  • 扫码书籍条码，查询价格区间
  • 将书籍按价格区间分类，输入各类重量（kg）
  • 自动计算各类小计和回收总价
  • 生成「回收确认单」并展示二维码，供学生扫码
  • 学生C端扫码确认回收后，管理员上传照片完成交易
🔁 交互复用：此处创建的确认单内容与C端C2-P06「站点直收确认单页」对接，交互逻辑与大宗回收模式一致。
🚚
B2-P07
待转运列表页
管理员查看所有待转运至仓库的打包书籍订单，统一管理转运任务，发起出库操作。
  • 列表展示所有待转运订单：订单号、来源（上门回收/站点直收/大宗）、书籍数量
  • Tab筛选：待转运 / 已转运
  • 批量选择订单，生成本次转运批次
  • 生成「转运入库码」供仓库管理员扫码
  • 展示本次转运汇总：上门N单/大宗M单/站点直收K单，合计金额
📊
B2-P08
转运出库码展示页
管理员将书籍运送至仓库时，出示此码供仓库人员扫码确认入库，完成转运闭环。
  • 居中展示大尺寸转运入库二维码
  • 展示本批次转运汇总信息（单数、来源类型、总书本数）
  • 仓库扫码确认后，页面更新为「入库确认成功」
  • 跳转至转运记录详情

B端 · 仓库管理员
仓库管理员负责扫码接收书籍入库、清点核验（标记异常）、拆包二次核验、书籍上架（生成货架编码并打印标签）。确保书籍从入库到上架的完整追溯体系。
仓库管理员操作流程
扫转运码接收
→
清点打包（标记异常）
→
完成入库
→
拆包二次核验
→
书籍上架（货架编码+打印）
📥
B3-P01
扫码入库接收页
仓管扫描站点管理员出示的转运入库码，系统调出本次转运的全部订单汇总信息，以批次维度展示（非逐本明细）。
  • 扫码后展示转运批次信息
  • 按回收模式汇总展示（非逐本）：上门回收 N单/共M本 / 大宗回收 K单/共X kg / 站点直收 P单/共Q本
  • 本次转运合计预估金额
  • 来源站点信息、转运人员
  • 「开始清点」按钮，进入清点操作页
🔍
B3-P02
清点核验 & 异常标记页
仓管清点各包装内书籍，发现异常时通过输入订单号进行标记（运损/数量不一致等），系统自动追溯责任人。清点完成后点击完成入库。
  • 订单包装清单展示（订单编号标注在包装上）
  • 逐包确认按钮（滑动确认/点击已清点）
  • 「标记异常」功能：输入订单号（或扫描包装条码）
  • 选择标记类型：运损 / 数量不一致 / 书籍缺失 / 其他
  • 上传异常照片
  • 系统根据标记类型自动关联责任人（回收员或站点管理员）
  • 清点完成后点击「完成入库」，书籍状态变为「已入库待核验」
⚖️ 责任追溯：标记异常后，系统根据订单的回收模式判断第一责任人（上门回收→回收员，站点直收→站点管理员），可在后台配置对应扣款规则。
📋
B3-P03
待核验书籍列表页
展示所有已入库但尚未完成拆包二次核验的书籍批次，仓管选择批次开始核验。
  • Tab筛选：待核验 / 核验中 / 已核验待上架
  • 每条批次展示：入库时间、书籍数量、来源站点
  • 点击进入拆包核验操作页
  • 异常标记批次高亮显示
🔎
B3-P04
拆包二次核验页
仓管拆包逐本核验书籍外观、新旧程度、笔记、正版等，发现问题可对订单进行标记追责，核验通过后书籍状态更新为「待上架」。
  • 展示批次书籍列表，逐本核验
  • 核验维度：书籍外观 / 新旧程度 / 笔记程度 / 是否正版
  • 「标记问题」：输入订单号，选择问题类型，上传照片
  • 系统根据问题类型执行处罚规则（如扣减打包核验人佣金）
  • 核验通过书籍：状态变为「待上架」
  • 问题书籍：单独处理（降级/退货/废弃）
  • 「完成核验」按钮，批次状态更新
📚
B3-P05
书籍上架操作页
仓管对核验通过的书籍进行上架操作，选择货架号后扫描书籍编码生成位置编码（如A-3-L-001），最终调用蓝牙热敏打印机打印标签贴至书脊。
  • 选择目标货架（下拉选择，显示货架号及剩余容量）
  • 货架剩余容量实时展示（如：A-3货架 剩余87/130位）
  • 扫描书籍编码，系统生成位置码（格式：货架-层-左右-序号，如A-3-L-001）
  • 生成编码后即时调用蓝牙热敏打印机打印标签
  • 货架满时提示「已达最大容量，请选择新货架」
  • 可添加新货架（输入货架号、设置最大容量）
  • 批量上架：连续扫码上架多本，每扫一本打印一张标签
  • 上架完成后书籍状态同步为「已上架」，后台更新货架位置信息
🖨️ 打印机连接：小程序需提前配对蓝牙热敏打印机，页面提供「连接打印机」入口，连接状态实时显示。打印标签内容：书籍位置码 + 书名简称 + 条形码。
🗄️
B3-P06
货架管理页
仓管查看所有货架的使用状态，管理货架容量配置，查看各货架当前存书情况。
  • 货架列表：货架号、最大容量、当前使用量、剩余量
  • 进度条直观展示容量使用率
  • 点击货架查看该货架内所有书籍列表（位置码+书名）
  • 「添加新货架」功能
  • 支持修改货架最大容量
📊
B3-P07
仓库工作台/首页
仓库管理员的主工作台，展示今日业务数据概览，提供快捷操作入口。
  • 数据统计：今日入库批次数 / 待核验数量 / 待上架数量 / 已上架总量
  • 快捷入口：「扫码入库」「去核验」「书籍上架」「货架管理」
  • 近期入库记录（最近5条）
  • 异常标记待处理提醒
